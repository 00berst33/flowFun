# revertPCAFlowSOM

Reverts elements of a FlowSOM object generated on principal components
back to channels.

## Usage

``` r
revertPCAFlowSOM(input, fsom, pca_obj, num_components)
```

## Arguments

- input:

  The aggregate flowFrame or data table used to generate the given
  FlowSOM object.

- fsom:

  A FlowSOM object as generated by
  [`clusterSubsetWithPCA()`](https://00berst33.github.io/flowFun/reference/clusterSubsetWithPCA.md).

- pca_obj:

  A [`prcomp`](https://rdrr.io/r/stats/prcomp.html) object as generated
  by [`doPCA()`](https://00berst33.github.io/flowFun/reference/doPCA.md)

- num_components:

  The number of principal components to use for reclustering, as
  determined by examining results of
  [`plotPCAScree()`](https://00berst33.github.io/flowFun/reference/plotPCAScree.md).

## Value

A FlowSOM object defined in terms of channels.

## Details

This function converts a FlowSOM object created from principal
components, as generated by
[`clusterSubsetWithPCA()`](https://00berst33.github.io/flowFun/reference/clusterSubsetWithPCA.md),
into an equivalent FlowSOM object defined in terms of channels instead.
The cluster/metacluster assignments for each cell remains the same, but
the resulting FlowSOM object is able to be used with built-in FlowSOM
functions like `PlotStars`, `PlotPies`, etc. Therefore other flowFun
functions like `plotLabeled2DScatter` and `searchByExpression` also work
on the resulting object.
